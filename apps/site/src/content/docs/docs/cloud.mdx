## Managed cloud, without changing the product model

The cloud version exists to remove operational overhead, not to simplify Stylora into a flatter product.

That means the hosted path should preserve the same core ideas:

- organizations stay first-class
- projects stay first-class
- PostgreSQL remains the initial backend focus

## Why a waitlist first

The goal is to shape the cloud offer around real teams instead of launching a generic pricing page too early.

If your team wants hosted access, join the [waitlist](/waitlist/).

## What to expect

- the same browser-native workflow
- less deployment overhead
- a cleaner path for teams that prefer operating fewer internal tools

## Scope discipline

Stylora does not claim support for every backend today. The first release is PostgreSQL first, with room for broader data backends over time once the core workspace is solid.
